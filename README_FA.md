# الگوریتم ژنتیک برای مسئله TSP

نسخه اصلی توضیحات پروژه در فایل `README.md` و به زبان انگلیسی قرار دارد. این فایل خلاصه فارسی همان توضیحات است.

## هدف پروژه

هدف برنامه حل مسئله فروشنده دوره‌گرد با استفاده از الگوریتم ژنتیک است. در این مسئله باید مسیری پیدا شود که:

- از یک شهر شروع کند
- از تمام شهرها دقیقاً یک بار عبور کند
- دوباره به شهر شروع برگردد
- کمترین طول ممکن را داشته باشد

برنامه روی دو ماتریس اجرا می‌شود:

1. ماتریس کامل و متقارن
2. ماتریس ناقص و نامتقارن

## کتابخانه‌های موردنیاز

برای نصب کتابخانه‌ها از دستور زیر استفاده می‌شود:

```bash
pip install numpy pandas matplotlib
```

کاربرد کتابخانه‌ها:

- `NumPy` برای محاسبات عددی و کار با ماتریس‌ها
- `Pandas` برای خواندن و ذخیره فایل‌های CSV
- `Matplotlib` برای رسم نمودارها

بقیه کتابخانه‌های استفاده‌شده جزو کتابخانه‌های استاندارد پایتون هستند و نصب جداگانه نمی‌خواهند.

## نحوه اجرا

برای تست اولیه:

```bash
python main.py --runs 2 --workers 1 --output test_results
```

برای اجرای نهایی با ۱۰۰ تکرار:

```bash
python main.py --runs 100 --workers 1 --output final_results
```

برای اجرای موازی:

```bash
python main.py --runs 100 --workers 4 --output final_results
```

در ویندوز می‌توان به‌جای `python` از `py` استفاده کرد.

## عملکرد کلی کد

در ابتدا فایل‌های CSV خوانده می‌شوند و به ماتریس فاصله تبدیل می‌شوند. هر جواب الگوریتم به‌صورت یک ترتیب از شماره شهرها نمایش داده می‌شود.

هزینه هر مسیر برابر مجموع فاصله بین شهرهای متوالی است. هرچه این مقدار کمتر باشد، جواب بهتر است.

جمعیت اولیه شامل ۱۰۰ مسیر است. در ماتریس کامل مسیرها تصادفی ساخته می‌شوند. در ماتریس ناقص برنامه تلاش می‌کند مسیرهایی تولید کند که از یال‌های موجود عبور کنند.

برای انتخاب والدها از انتخاب تورنمنتی استفاده شده است. برای ترکیب مسیرها از Ordered Crossover استفاده می‌شود تا هیچ شهری حذف یا تکرار نشود.

سه نوع جهش در برنامه وجود دارد:

- جابه‌جایی دو شهر
- معکوس کردن بخشی از مسیر
- به‌هم‌ریختن تصادفی بخشی از مسیر

برای ماتریس ناقص، یال‌های ناموجود جریمه بزرگی دریافت می‌کنند تا الگوریتم آن مسیرها را انتخاب نکند.

## خروجی‌ها

برنامه فایل‌های CSV برای نتایج خام و خلاصه، فایل JSON برای بهترین مسیرها و نمودارهای PNG برای مقایسه پارامترها تولید می‌کند.

مهم‌ترین ستون‌های خروجی عبارت‌اند از:

- میانگین طول مسیر
- انحراف معیار
- بهترین طول مسیر
- نسل همگرایی
- درصد مسیرهای معتبر

## بررسی صحت اجرا

برای درست بودن نتیجه باید:

- هر شهر دقیقاً یک بار در مسیر باشد
- هیچ شهری تکرار نشود
- طول مسیر `inf` یا `NaN` نباشد
- درصد مسیر معتبر نزدیک به ۱۰۰ باشد
- نمودار برازندگی در طول نسل‌ها کاهش پیدا کند و سپس تقریباً ثابت شود

چون الگوریتم ژنتیک تصادفی است، هر آزمایش ۱۰۰ بار با بذرهای مختلف اجرا می‌شود و میانگین و انحراف معیار گزارش می‌شود.
