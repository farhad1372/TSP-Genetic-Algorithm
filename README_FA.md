# الگوریتم ژنتیک برای مسئله TSP

نسخه اصلی توضیحات پروژه در فایل `README.md` و به زبان انگلیسی قرار دارد. این فایل نسخه فارسی توضیحات پروژه است.

## هدف پروژه

هدف برنامه حل مسئله فروشنده دوره‌گرد یا TSP با استفاده از الگوریتم ژنتیک است.

در این مسئله باید مسیری پیدا شود که:

- از یک شهر شروع کند
- از تمام شهرها دقیقاً یک بار عبور کند
- دوباره به شهر شروع برگردد
- کمترین طول ممکن را داشته باشد

هر کروموزوم یک ترتیب کامل از شهرها را نشان می‌دهد.

برنامه روی دو ماتریس اجرا می‌شود:

1. ماتریس کامل و متقارن
2. ماتریس ناقص و نامتقارن

## امکانات برنامه

- حل مسئله TSP با الگوریتم ژنتیک
- انتخاب تورنمنتی
- ترکیب Ordered Crossover
- جهش Swap، Inversion و Scramble
- نخبگی
- مقایسه نرخ‌های مختلف جهش
- مقایسه نرخ‌های مختلف ترکیب
- مقایسه نرخ‌های مختلف نخبگی
- مقایسه روش‌های جایگزینی `mu + lambda` و `mu, lambda`
- پشتیبانی از ماتریس ناقص و نامتقارن
- رسم نمودار برازندگی
- ذخیره نتایج در فایل‌های CSV و JSON
- اجرای چندباره با بذرهای تصادفی متفاوت

## پیش‌نیازها

استفاده از Python 3.9 یا نسخه‌های جدیدتر پیشنهاد می‌شود.

کتابخانه‌های موردنیاز در فایل `requirements.txt` قرار دارند.

برای نصب تمام وابستگی‌ها دستور زیر را اجرا کنید:

```bash
pip install -r requirements.txt
```

کتابخانه‌های اصلی پروژه عبارت‌اند از:

- NumPy
- Pandas
- Matplotlib

سایر ماژول‌های استفاده‌شده جزو کتابخانه استاندارد پایتون هستند و به نصب جداگانه نیاز ندارند.

## ساختار فایل‌های پروژه

فایل‌های زیر باید در یک پوشه قرار بگیرند:

```text
project/
|-- main.py
|-- requirements.txt
|-- distance_matrix_symmetric.csv
|-- distance_matrix_incomplete_asymmetric.csv

```

## نحوه اجرا

ابتدا بهتر است برنامه با تعداد اجرای کم آزمایش شود:

```bash
python main.py --runs 2 --workers 1 --output test_results
```

برای اجرای نهایی با ۱۰۰ تکرار مستقل:

```bash
python main.py --runs 100 --workers 1 --output final_results
```

برای استفاده از چند هسته پردازنده:

```bash
python main.py --runs 100 --workers 4 --output final_results
```

در ویندوز می‌توان به‌جای `python` از `py` استفاده کرد:

```bash
py main.py --runs 100 --workers 1 --output final_results
```

## ورودی‌های خط فرمان

- `--symmetric`: مسیر فایل ماتریس کامل و متقارن
- `--asymmetric`: مسیر فایل ماتریس ناقص و نامتقارن
- `--runs`: تعداد اجرای مستقل برای هر حالت
- `--workers`: تعداد پردازش‌های هم‌زمان
- `--output`: پوشه ذخیره نتایج

نمونه اجرا:

```bash
python main.py   --symmetric "distance_matrix_symmetric.csv"   --asymmetric "distance_matrix_incomplete_asymmetric.csv"   --runs 100   --workers 2   --output final_results
```

## نحوه کار برنامه

### خواندن ماتریس فاصله

تابع `load_distance_matrix` فایل CSV را می‌خواند و آن را به یک ماتریس NumPy تبدیل می‌کند. این تابع همچنین بررسی می‌کند که ماتریس مربعی باشد.

### نمایش مسیر

هر مسیر به‌صورت آرایه‌ای از شماره شهرها نگهداری می‌شود.

مثال:

```text
[0, 4, 2, 1, 3]
```

هنگام محاسبه هزینه، شهر آخر به‌صورت خودکار به شهر اول متصل می‌شود.

### محاسبه برازندگی

مقدار برازندگی برابر طول کل مسیر است. چون هدف کمینه کردن فاصله است، مقدار کمتر نشان‌دهنده جواب بهتر است.

در ماتریس ناقص، به یال‌های ناموجود جریمه بزرگی داده می‌شود تا مسیرهای نامعتبر به‌عنوان جواب مناسب انتخاب نشوند.

### تولید جمعیت اولیه

برای ماتریس کامل، مسیرها به‌صورت جایگشت‌های تصادفی ساخته می‌شوند.

برای ماتریس ناقص، برنامه تلاش می‌کند مسیرهایی تولید کند که از یال‌های موجود استفاده کنند. این کار باعث می‌شود تعداد مسیرهای معتبر در جمعیت اولیه بیشتر باشد.

### انتخاب والدین

در انتخاب تورنمنتی، چند کروموزوم به‌صورت تصادفی انتخاب می‌شوند و بهترین عضو آن‌ها به‌عنوان والد در نظر گرفته می‌شود.

### عملگر ترکیب

از Ordered Crossover استفاده شده است. این روش باعث می‌شود هیچ شهری حذف یا تکرار نشود.

### عملگرهای جهش

برنامه سه نوع جهش دارد:

- Swap: جابه‌جایی دو شهر
- Inversion: معکوس کردن بخشی از مسیر
- Scramble: به‌هم‌ریختن تصادفی بخشی از مسیر

جهش باعث حفظ تنوع جمعیت می‌شود و احتمال همگرایی زودرس را کاهش می‌دهد.

### روش‌های جایگزینی

سه روش جایگزینی در برنامه وجود دارد:

- جایگزینی نسلی همراه با نخبگی
- `mu + lambda`: انتخاب نسل بعد از بین والدها و فرزندان
- `mu, lambda`: انتخاب نسل بعد فقط از بین فرزندان

### شرط توقف

الگوریتم در دو حالت متوقف می‌شود:

- رسیدن به حداکثر تعداد نسل‌ها
- پیدا نشدن جواب بهتر برای تعداد مشخصی نسل متوالی

## آزمایش‌ها

در برنامه اثر موارد زیر بررسی می‌شود:

- نوع جهش
- درصد جهش
- درصد ترکیب

## بررسی صحت نتایج

برای اطمینان از درست بودن اجرا باید موارد زیر بررسی شوند:

- همه شهرها دقیقاً یک بار در مسیر وجود داشته باشند
- هیچ شهری تکرار نشده باشد
- طول مسیر `inf` یا `NaN` نباشد
- مقدار `valid_route_percent` نزدیک به ۱۰۰ یا برابر ۱۰۰ باشد
- نمودار برازندگی به‌طور کلی کاهش پیدا کند و سپس ثابت شود
- اجرای دوباره با بذرهای یکسان نتایج یکسانی تولید کند
